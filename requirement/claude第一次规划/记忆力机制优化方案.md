## S - Self-Evolving-Baby (记忆系统)

### 当前问题诊断

| 问题 | 现状 | 影响 |
|------|------|------|
| 记忆压缩无持久化 | Token 压缩后仅存内存 | 重启后丢失压缩上下文 |
| 三层记忆未实现 | 只有 memory.json 索引 | 无法区分短期/中期/长期 |
| 记忆检索原始 | 全文匹配，无语义搜索 | 检索效率低 |
| 遗忘机制缺失 | 无选择性遗忘 | 记忆无限增长 |

## 一、架构设计

### 1.1 三层记忆模型

```
┌──────────────────────────────────────────────────────────────────┐
│                        三层记忆架构                                │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  【短期记忆】 LLM Context Window                                 │
│  ├── 工作消息（当前会话）                                         │
│  ├── 活跃任务状态                                                │
│  └── 压缩后的历史摘要                                            │
│      ↓ 触发条件：Token > 60% 或 手动调用                          │
│                                                                  │
│  【中期记忆】 Session Storage                                     │
│  ├── 世代索引 (memory.json)                                      │
│  ├── 压缩上下文快照 (archives/compressed_{gen}_{time}.json)        │
│  ├── 关键决策记录 (archives/decisions_{gen}.jsonl)                │
│  └── 工具使用统计 (archives/tool_stats_{gen}.json)                 │
│      ↓ 触发条件：世代推进或 Token 超限                            │
│                                                                  │
│  【长期记忆】 Persistent Storage                                   │
│  ├── 核心智慧 (core_wisdom.md)                                   │
│  ├── 能力画像 (skills_profile.json)                               │
│  ├── 模式库 (pattern_library.json)                                │
│  └── 知识图谱 (knowledge_graph.json)                              │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 八、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 向量存储占用过大 | 存储成本高 | 定期清理低价值索引 |

| 遗忘机制误删重要记忆 | 不可逆损失 | 多重确认 + 回收站 |
