# 提示词自主动态拼装实现方案

**版本：** v1.0
**日期：** 2026-04-18
**类型：** 提示词系统增强

---

## 一、目标

1. **记忆选择部分先维持为空** — `MEMORY` 组件默认不参与提示词拼装
2. **先实现提示词自主动态拼装** — Agent 可通过 `<active_components>` XML 标签动态控制提示词组件组成

---

## 二、设计方案

### 2.1 组件注册表

`PromptManager` 已包含以下 11 个组件：

| name | priority | required | 默认激活 |
|------|----------|----------|---------|
| SOUL | 10 | True | ✅ |
| TASK_CHECKLIST | 20 | False | ❌ |
| CODEBASE_MAP | 30 | False | ❌ |
| DYNAMIC | 40 | False | ❌ |
| IDENTITY | 50 | False | ❌ |
| AGENTS | 60 | True | ✅ |
| USER | 70 | False | ❌ |
| MEMORY | 80 | False | ❌（本次修改） |
| current_rules | 85 | False | ✅ |
| TOOLS_INDEX | 90 | False | ❌ |
| ENV_INFO | 100 | False | ❌ |

### 2.2 动态切换机制

```
Agent 回复 → <active_components>SOUL, AGENTS, MEMORY</active_components>
              ↓
        ResponseParser._extract_active_components()
              ↓
        agent.py: pm.select_components(["SOUL", "AGENTS", "MEMORY"])
              ↓
        _active_components_override = ["SOUL", "AGENTS", "MEMORY"]
              ↓
        _select_components() 读取 override → 后续 build() 使用新组件集
```

### 2.3 优先级

1. `build(include=[...])` 参数优先
2. `_active_components_override` 次之（LLM 标签驱动）
3. 默认 `["SOUL", "AGENTS", "current_rules"]`（不包含 MEMORY）
