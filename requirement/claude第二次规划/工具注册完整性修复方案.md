## 问题背景

通过 `tests/prompt_debugger.py` 提示词打靶测试发现：

Agent 从 SOUL.md / AGENTS.md 文档中列举出 34 个工具，但实际注册到 LLM 的只有 14 个（原来的 Key_Tools）。这导致：
- SOUL.md 核心 SOP 断裂（`commit_compressed_memory_tool` 等无法调用）
- AGENTS.md 工具速查表几乎全部失效
- Agent 产生"工具幻觉"，列举了根本不存在的工具名

## 修复方案

### Phase 1: 实现缺失工具
- `write_dynamic_prompt_tool`：全量覆盖写入 DYNAMIC.md（不同于 add_insight 的追加）

### Phase 2: 注册所有未绑定工具
- 将上述 15 个工具的 LangChain `@tool` 包装加入 `tools/Key_Tools.py`
- 统一工具名后缀为 `_tool`

### Phase 3: 清理文档
- 删除 AGENTS.md 中 4 个不存在的工具引用
- 统一工具名为实际注册名（加 `_tool` 后缀）
- 补充新注册工具到速查表
