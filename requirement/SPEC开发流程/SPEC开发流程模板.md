# SPEC 开发流程模板

**版本：** v1.0
**日期：** 2026-04-18
**用途：** 虾宝自我进化系统的标准化开发流程

---

## 概述

### 目的

本模板为虾宝自我进化系统提供标准化的开发流程，确保每次开发任务都有完整的需求分析、设计、实现、测试、文档归档和质量门控。

### 适用范围

- 所有 Phase 1-8 的功能开发
- Bug 修复和优化
- 重构和模块拆分
- 提示词相关变更

### 核心原则

1. **Phase 递进**：每轮开发完成后，Phase 状态递增
2. **规划-实施-归档**：规划文档和任务报告分离存放，供 AI 查询
3. **强制质量门控**：测试覆盖、代码审查、提示词打靶（如涉及）
4. **诚实标注状态**：✅ 完整 / ⚠️ 框架 / ❌ 未实现

## 质量门控

### 代码审查清单

- [ ] **命名规范**：模块文件 snake_case，测试文件 test_*.py，类 PascalCase，函数 snake_case
- [ ] **类型注解**：公开函数/方法必须有类型注解
- [ ] **文档字符串**：类和公开函数必须有中英文文档字符串
- [ ] **无空壳模块**：禁止创建无实际逻辑的占位模块 ⚠️
  - 空壳定义：仅有 `pass` 或 `...` 的类/函数
  - 允许情况：抽象基类框架（需标注 ⚠️ 框架）
- [ ] **无硬编码**：配置值必须从 config.toml 读取，禁止硬编码路径/密钥
- [ ] **向后兼容**：修改公开 API 时保留向后兼容接口

### 测试覆盖

- [ ] 单元测试通过率 100%
- [ ] 集成测试通过率 100%
- [ ] 覆盖率 >= 80%（新模块必须）
- [ ] 所有 pytest 命令实际运行通过

### 提示词打靶（如涉及）

- [ ] 相关测试用例通过
- [ ] 工具名与注册名一致（对照 tools/Key_Tools.py）
- [ ] SOUL.md / AGENTS.md 规则验证通过

## 使用说明

1. **开发前**：
   - 填写 "Phase {N}" 各节
   - 将规划文档存入 `requirement/`

2. **开发中**：
   - 按 "测试" 节执行测试
   - 确保所有 "质量门控" 检查项通过

3. **开发后**：
   - 将任务报告存入 `report_history/`
   - 执行 Git 提交

4. **联动更新**：
   - 完成后更新 INDEX.md 相关章节
   - 确保 Phase 状态正确递增

## 7. 流程复盘与优化（每轮结束后执行）

> **⚠️ 重要**：本模板在每轮开发结束后，由 AI 工具进行复盘总结，持续改进流程。

### 7.1 复盘时机

每轮开发流程完成后（Git 提交后），AI 工具应自动触发复盘，填写以下内容：

**实际耗时：** {估算 vs 实际对比}
**完成质量：** {按计划/超预期/未达预期}

**流程执行情况：**

| 流程步骤 | 计划时间 | 实际时间 | 偏差 | 问题 |
|----------|----------|----------|------|------|
| 需求分析 | {计划} | {实际} | {偏差} | {问题} |
| 设计 | {计划} | {实际} | {偏差} | {问题} |
| 实现 | {计划} | {实际} | {偏差} | {问题} |
| 测试 | {计划} | {实际} | {偏差} | {问题} |
| 文档归档 | {计划} | {实际} | {偏差} | {问题} |

### 7.2 改进触发条件

| 触发条件 | 优先级 | 说明 |
|----------|--------|------|
| 某步骤频繁出现问题 | P1 | 必须改进 |
| 某步骤耗时超出预期 50% | P2 | 建议改进 |
| AI 发现更好的方法 | P3 | 可选改进 |
| 遇到反复出现的同类问题 | P1 | 必须改进 |

### 7.3 模板更新规则

当复盘发现需要改进时，由 AI 工具决定是否更新本模板：

**必须更新模板的情况：**
- 某流程步骤被反复跳过或忽略
- 发现系统性遗漏（如总是忘记更新 INDEX.md）
- 质量门控存在漏洞

**更新流程：**
1. AI 在复盘时提出改进建议
2. 下轮开发前，由 AI 判断是否采纳
3. 如采纳，更新本模板并记录到变更日志
4. 如不采纳，在复盘记录中说明原因

### 7.4 流程健康度指标

| 指标 | 定义 | 目标值 | 监控方式 |
|------|------|--------|----------|
| 规划准确率 | 实际交付与规划的符合度 | >= 90% | 复盘记录统计 |
| 测试覆盖率 | 有测试的模块占比 | >= 80% | pytest --cov |
| 文档完备率 | 按时提交报告的比例 | 100% | report_history/ 检查 |
| Bug 遗留率 | 上线后发现的 Bug 数 | <= 5% | 后续追踪 |
| 流程偏差率 | 实际流程与模板的偏差 | <= 10% | 复盘自评 |
