# 虾宝自我进化系统 - 完整规划

## 一、项目愿景

**虾宝 (Xiaobao)** 是一个能够持续自我进化的 AI Agent 系统。它能够在每次"苏醒-执行-休眠"循环中：

1. **分析自身状态** → 发现瓶颈和不足
2. **制定进化计划** → 规划改进方向
3. **执行代码修改** → 实际改进自身能力
4. **验证迭代效果** → 确保每次进化都比之前更优秀
5. **保存进化记忆** → 将经验沉淀到知识库中

最终目标：虾宝成为一个**能够自主学习、自主规划、自主改进**的通用智能体。

### 5.1 代码修改策略选择器

虾宝需要根据场景选择合适的修改方式：

```python
class CodeModificationStrategy:
    """代码修改策略选择器"""

    STRATEGIES = {
        "diff_edit": {
            "适用场景": "小范围修改（< 50 行），保留上下文",
            "示例": "修复 Bug、优化单函数、修改常量"
        },
        "file_rewrite": {
            "适用场景": "大幅重构、结构变化",
            "示例": "提取函数、重写模块、优化算法"
        },
        "modular_plugins": {
            "适用场景": "添加新功能、解耦模块",
            "示例": "新增工具、提取工具类、模块拆分"
        },
        "code_generation": {
            "适用场景": "创建新文件、全新模块",
            "示例": "新增测试文件、添加配置文件、创建新工具"
        }
    }

    def select_strategy(self, task_type, scope, risk_level):
        """根据任务特征选择策略"""
        ...
```

### 5.2 进化安全策略（分阶段）

**阶段 1：快速迭代期**
- ✅ 每次修改后语法检查
- ✅ 简单功能验证（ smoke test）
- ⚠️ 不阻塞进化，但记录风险

**阶段 2：稳健进化期**
- ✅ 语法检查 + 关键路径测试
- ✅ 代码复杂度监控（确保不增加）
- ✅ 自动回滚机制（如果测试失败）

**阶段 3：质量保证期**
- ✅ 完整测试套件（>80% 覆盖率）
- ✅ 代码质量门禁（复杂度、重复率）
- ✅ 性能基准测试（确保不退化）
- 🔒 任意一项不通过则阻止进化

### 5.3 知识图谱设计

```python
class KnowledgeGraph:
    """
    知识图谱：存储实体和关系
    实体：文件、函数、类、工具、任务、Bug、Feature
    关系：依赖、调用、修改、解决、实现
    """
    # 示例图谱结构
```

## 八、风险与应对

### 风险 1：进化导致系统崩溃

**概率：** 中 | **影响：** 严重

**应对：**
- ✅ 每次重启前运行完整测试
- ✅ 自动备份（每 10 分钟 + 每次重启前）
- ✅ 回滚机制（测试失败自动回滚）
- ✅ 逐步验证（先语法，再单元测试，再集成测试）

### 风险 2：无限循环进化

**概率：** 中 | **影响：** 高

**应对：**
- ✅ 进化次数上限（每阶段最多 20 代）
- ✅ 收敛检测（如果连续 3 代无改进，停止）
- ✅ 人工审核（关键阶段需要人工确认）

**概率：** 低 | **影响：** 中

- ✅ 功能清单检查（确保功能完整）

### 风险 4：LLM 理解偏差

**概率：** 高 | **影响：** 中

**应对：**
- ✅ 提供详细示例
- ✅ 分步骤执行（每步验证）

## 十一、FAQ

**Q1：虾宝会修改自己的代码吗？会不会把自己改坏？**
> A：会，但有多层防护。每次修改后必须通过语法检查，关键修改需通过测试。如果测试失败会自动回滚。同时每次重启前会备份。

**Q2：进化需要人工审核吗？**
> A：早期需要，后期逐步减少。我们建议：里程碑级别的变更需要人工确认，日常小优化可以自动进行。

**Q3：虾宝会忘记之前的能力吗？**
> A：不会。世代之间会继承核心记忆（`core_wisdom`），能力会逐渐积累而非重置。能力画像系统会追踪所有能力。

**Q4：如果虾宝陷入死循环怎么办？**
> A：有多重保护：任务清单拦截（未完成任务禁止重启）、世代上限（每阶段最多 20 代）、收敛检测（3 代无改进则停止）。

**Q5：如何验证虾宝真的变强了？**
> A：通过能力画像的量化评分、代码质量指标（复杂度、覆盖率）、以及实际任务完成度的对比。每个世代都会生成进化报告。
