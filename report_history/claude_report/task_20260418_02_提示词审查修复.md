## 发现的问题

### 问题 1：工具名称不一致（严重）

提示词中使用的工具名与实际注册到 LLM 的名称存在差异。

| 提示词中的错误名称 | 实际工具名 |
|---|---|
| `check_python_syntax_tool` | `check_python_syntax` |
| `get_agent_status` | `self_test`（不存在，应替换为 self_test） |
| `execute_shell_command` | `cli_tool`（主命令工具） |
| `list_file_entities` | `list_file_entities_tool` |
| `get_code_entity` | `get_code_entity_tool` |
| `search_and_read_tool` | `search_and_read` |
| `find_definitions_tool` | `find_definitions` |
| `find_function_calls_tool` | `find_function_calls` |
| `search_imports_tool` | `search_imports` |
| `set_plan` | `set_plan_tool` |
| `tick_subtask` | `tick_subtask_tool` |
| `commit_compressed_memory` | `commit_compressed_memory_tool` |
| `trigger_self_restart` | `trigger_self_restart_tool` |
| `apply_diff_edit` | `apply_diff_edit_tool` |

### 问题 2：内容冲突（严重）

SOUL.md 顶部写"禁止修改 SOUL.md 和 AGENTS.md"，但底部"提示词文件结构"章节又说 workspace 里的同名文件"可覆盖"，自相矛盾。

### 问题 3：Windows 兼容性问题

SOUL.md 示例中使用 `cat agent.py`，在 Windows 系统上无法使用。

### 问题 4：禁止删除列表不完整

SOUL.md 禁止删除列表缺少 `core/core_prompt/AGENTS.md`。

### 问题 5：路径引用过时

AGENTS.md 中提示词路径引用 `prompts/` 而非 `workspace/prompts/`。
