## 机制说明

### 工作原理

```
Agent 回复 → <active_components>SOUL, AGENTS, MEMORY</active_components>
              ↓
        ResponseParser 解析标签
              ↓
        agent.py 调用 pm.select_components(["SOUL", "AGENTS", "MEMORY"])
              ↓
        _active_components_override 生效
              ↓
        后续 pm.build() 自动使用新组件集
```

### 使用示例

```xml
<thinking>需要查看历史记忆来完成跨代任务</thinking>
<active_components>SOUL, AGENTS, MEMORY, current_rules</active_components>
```

### 可用组件

| 组件 | 默认激活 | 说明 |
|------|---------|------|
| SOUL | ✅ | 核心使命（不可修改） |
| AGENTS | ✅ | 操作规范 SOP |
| current_rules | ✅ | 激活规则集 |
| MEMORY | ❌ | 记忆上下文（需主动激活） |
| TASK_CHECKLIST | ❌ | 任务清单 |
| DYNAMIC | ❌ | 动态提示词区域 |
| IDENTITY | ❌ | 身份定义 |
| USER | ❌ | 用户信息 |
| CODEBASE_MAP | ❌ | 代码库认知地图 |
| TOOLS_INDEX | ❌ | 工具手册索引 |
| ENV_INFO | ❌ | 环境信息 |
