## 任务概述

为虾宝 Agent 实现 OpenClaw 风格的 Skill 自我扩展系统，支持 Agent 自主添加、修改、优化、删除工具能力。

**背景**：Agent 需要动态扩展自身能力，当前只能依赖固定工具集，无法自我进化工具列表。

**目标**：
1. 建立 Skill 发现/加载/执行机制
2. Agent 可自主管理 Skill（install/update/optimize/uninstall）
3. Skill 存储在 `workspace/skills/`，Agent 可读写
4. 复用已有 CompositeTool 系统

## 遇到的问题

### 1. SKILL.md 解析失败（已修复）

**问题**：`## parameters` section 被 `stripped.startswith('#')` 误判为注释跳过，导致所有参数解析失败。

**原因**：`##` 格式的章节头需要优先于 `#` 注释判断。

**解决**：将章节头检测提前到注释过滤之前，并修复 `header_content == 'parameters'` 中的空格问题。

### 2. 缩进解析歧义（已修复）

**问题**：`## parameters` 后的空行导致 `param_base_indent=0`，使第一个参数的缩进被误判。

**原因**：缩进比较逻辑在空行情况下失效。

**解决**：改用 `current_indent <= section_indent` 判断（允许同级），并动态跟踪 section indent。

### 3. `continue` 导致参数丢失（已修复）

**问题**：`if in_params_section:` 块的 `continue` 导致最后的参数保存逻辑被跳过。

**原因**：`elif` 分支导致最后一个 `if current_param` 保存逻辑被跳过。

**解决**：移除 `continue`，让所有分支都能到达最后的参数保存逻辑。

### 4. Pydantic v2 schema 构建（已修复）

**问题**：Pydantic v2 中 `type(name, (BaseModel,), {name: (Type, Field(...))})` 语法报错。

**原因**：Pydantic v2 不再支持此语法。

**解决**：使用 `__annotations__` namespace 方式：`type(name, (BaseModel,), {"__annotations__": {...}, "field": Field(...)})`。
