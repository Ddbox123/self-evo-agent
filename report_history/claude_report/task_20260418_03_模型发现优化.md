## 任务概述

agent.py 启动时无论使用什么 provider 都会尝试动态发现模型参数，导致云端 API（如 MiniMax）每次启动都白白等待 5 秒超时，降低启动速度。优化为仅本地模型（local/ollama）启用动态发现，云端模型直接跳过。

### 核心改动（agent.py 第 400-407 行）

```python
# 修改前
if getattr(self.config.llm_discovery, 'enabled', True):
    discovery = ModelDiscovery(...)

# 修改后
provider = getattr(self.config.llm, 'provider', '')
is_local_provider = provider in ('local', 'ollama')
discovery_enabled = getattr(self.config.llm_discovery, 'enabled', True) and is_local_provider

if discovery_enabled:
    discovery = ModelDiscovery(...)
```

### 逻辑说明

- **本地模型**（`local`、`ollama`）：context_window 取决于本地部署配置，可能不确定 → 动态发现
- **云端模型**（`minimax`、`openai`、`deepseek` 等）：context_window 已在文档中明确 → 直接使用配置文件值
