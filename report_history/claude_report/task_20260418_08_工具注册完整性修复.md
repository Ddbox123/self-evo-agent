### write_dynamic_prompt_tool 失败原因与修复

**原因**：SOUL.md 世代任务流程中未说明 `write_dynamic_prompt_tool` 的使用场景，模型选择了语义更接近的 `read_file_tool`。

**修复**：已在 SOUL.md 世代任务流程中添加 `write_dynamic_prompt_tool` 的示例，并在 AGENTS.md 速查表中补充该工具。

### ToolExecutor 工具名映射修复（关键 Bug）

打靶测试发现虽然 `Key_Tools.py` 的 LangChain Tool 注册了 `_tool` 后缀名，但 `ToolExecutor._tool_map` 中 memory 工具只有无后缀名（如 `"read_dynamic_prompt"`），LLM 调用带 `_tool` 后缀的同名工具时报告"未知工具"。

**根因**：`ToolExecutor._register_default_tools()` 中只导入了工具函数，但映射表只有无后缀名，部分工具（如 `get_memory_summary_tool`、`write_dynamic_prompt_tool`）甚至未导入。

**修复**：在 `core/infrastructure/tool_executor.py` 中：
1. 补充所有 memory 工具（`get_memory_summary_tool`、`write_dynamic_prompt_tool`、`clear_generation_task`）的导入
2. 给所有 memory 工具和 shell 工具补充双重映射（无后缀 + `_tool` 后缀），确保 LLM 调用的任何命名变体都能找到

修复后 ToolExecutor 共注册 99 个工具映射，所有 17 个核心工具均可在 `_tool_map` 中找到。
