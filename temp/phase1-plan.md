## 📝 任务清单

### 任务 1: 架构重构

**现状问题**:
- `agent.py` 1925 行，过于庞大
- 工具执行逻辑与 Agent 核心循环耦合
- 职责不清晰

**目标结构**:
```
agent.py (精简至 ~300 行)
    │
    ├──► core/agent_core.py (Agent 核心循环)
    ├──► core/tool_executor.py (工具执行器，已有部分实现)
    ├──► core/event_handlers.py (事件处理器)
    └──► tools/ (保持现有结构)
```

**步骤**:
- [ ] 1.1 提取 Agent 核心循环到 `core/agent_core.py`
- [ ] 1.2 完善 `core/tool_executor.py` (已有基础)
- [ ] 1.3 创建 `core/event_handlers.py` (事件处理器)
- [ ] 1.4 重构 `agent.py` 为入口文件
- [ ] 1.5 运行测试验证

**完成标准**:
- `agent.py` 行数 < 500 行
- 各模块职责清晰
- 所有测试通过

### 任务 2: 安全加固

**现状问题**:
- 使用黑名单拦截危险命令
- 路径验证基于模式匹配
- 可能被绕过

**目标方案**:
- 白名单机制：只允许明确的命令模式
- 路径沙箱：限制在项目目录内
- 命令参数验证

**步骤**:
- [ ] 2.1 创建 `core/security.py` (安全策略模块)
- [ ] 2.2 实现命令白名单
- [ ] 2.3 实现路径沙箱验证
- [ ] 2.4 更新 `shell_tools.py` 使用新安全模块
- [ ] 2.5 添加安全测试用例

**完成标准**:
- 所有 shell 命令必须通过白名单验证
- 路径访问限制在项目目录内
- 安全测试覆盖所有危险操作
