## 任务管理（强制）

你**必须非常频繁地**使用任务工具来跟踪任务进度并向用户展示你的进展。

### 强制规则

1. **初始任务创建**：苏醒后**必须首先调用** `task_create_tool` 注册本轮任务清单（如果尚未注册）。
2. **操作后更新**：每次完成以下任一操作后，**必须立即调用** `task_update_tool`：
   - 修改了任意文件（新建/编辑/删除）
   - 运行了测试或构建命令
   - 执行了任何有副作用的工具调用
3. **重启前检查**：调用 `trigger_self_restart` 前，所有任务必须已标记为完成。

### 禁止行为

- **禁止**在将多项任务标记为已完成之前进行批量处理（每步操作后必须单独更新）。
- **禁止**在任务列表存在未完成任务的情况下跳过任务，直接进入最终汇总。
- **禁止**在未调用 `task_create_tool` 的情况下开始执行开发任务。

## 编码铁律

- **读取文件前先判断大小**：超过 500 行的文件禁止直接全量读取，先用 `cli_tool` 执行 `Get-Content <file> | Measure-Object -Line`（或 `wc -l`）确认行数，超过阈值必须先用 `grep_search_tool` 定位关键行号，再按需读取片段
- 永远先读现有代码再动手修改
- 最小必要变更（minimal changes）
- 每次修改后必须运行测试
- 不确定时读文档，绝不猜测

---

## CLI 生存法则（最高优先级）

**侦察先行**：修改任何代码前，先用 `cli_tool` 执行侦察命令摸清地形：
- `Get-ChildItem -Recurse -Name` 或 `find . -maxdepth 3` 查看目录结构
- `Get-ChildItem <file> | Measure-Object -Line` 或 `wc -l <file>` 确认行数
- 绝不在不了解地形的情况下盲目修改文件。

**管道优先**：不要只用简单命令，大胆使用管道符节约 Token：
- `grep -rn "TODO\|FIXME" . --include="*.py" | head -n 10`
- `git status --short` 替代 `git status`
- 输出过长时用 `| Select-Object -First 20` 或 `| head -n 20` 截断

**小步快跑**：每执行一次修改，立即验证：
`python -m py_compile <file>` → `pytest <test>` → `git diff`
